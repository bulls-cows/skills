# 代码质量检查与审核助手

执行代码质量检查和代码审核，包括 lint 检查、语法错误、逻辑错误、安全漏洞和最佳实践检查。

## 功能特性

- 检查 `package.json` 中是否存在 `lint:fix` 或 `lint` 脚本
- 优先使用 `npm run lint:fix` 命令，其次使用 `npm run lint`
- 执行 lint 进行代码风格和类型检查
- 审核改动文件的语法错误、逻辑错误、安全漏洞和最佳实践

## 使用方式

直接调用 `/review` 即可执行代码质量检查和审核。

## 工作流程

### 阶段一：检查 lint 脚本可用性

1. 检查项目根目录下是否存在 `package.json` 文件
2. 如果存在，读取 `package.json` 中的 `scripts` 字段
3. 检查是否定义了 `lint:fix` 或 `lint` 命令（优先 `lint:fix`）

**情况 1：package.json 不存在或无 lint 相关脚本**

- 跳过 lint 检查
- 直接进入阶段三（代码审核）

**情况 2：存在 lint 相关脚本**

- 记录要执行的命令（`npm run lint:fix` 或 `npm run lint`）
- 检查 Node.js 版本是否满足 `.nvmrc` 要求
- 如果版本不满足，显示警告并跳过 lint 检查
- 如果版本满足，继续执行阶段二

### 阶段二：执行 lint 检查

执行阶段一确定的命令（`npm run lint:fix` 或 `npm run lint`）检测代码风格和 TypeScript 类型。

**情况 1：lint 通过，无错误**

- 显示成功消息
- 继续执行阶段三

**情况 2：lint 发现错误**

- 显示错误信息
- 尝试自动修复错误
- 修复后继续执行阶段三

### 阶段三：获取改动文件列表

1. 使用 git 命令获取所有改动的文件：

   ```bash
   git diff --name-only HEAD
   ```

   或者对于暂存的文件：

   ```bash
   git diff --cached --name-only
   ```

2. 过滤出代码文件（排除自动生成的文件、配置文件等）

### 阶段四：代码审核

对于每个改动的文件，进行以下检查：

1. **语法错误检查**
   - 检查是否有明显的语法错误
   - 检查是否有未闭合的标签
   - 检查是否有未定义的变量

2. **逻辑错误检查**
   - 检查是否有空指针引用
   - 检查是否有数组越界
   - 检查是否有逻辑判断错误

3. **安全漏洞检查**
   - 检查是否有 XSS 风险
   - 检查是否有敏感信息泄露
   - 检查是否有 SQL 注入风险（如果有模板字符串拼接 SQL）

4. **最佳实践检查**
   - 检查是否有 console.log/debugger 等调试代码（**例外：在 catch 块中的 console.warn 不视为问题**）
   - 检查是否有硬编码的敏感信息
   - 检查是否有不必要的注释或死代码

### 阶段五：输出审核结果

如果发现问题，按照以下格式输出：

```markdown
## 审核结果

### 文件: [filename]

发现以下问题：

1. **[严重/中等/轻微]** - [问题描述]
   - 位置: [行号]
   - 建议: [修复建议]
```

**审核结果判断：**

- 如果没有发现问题 → 输出审核通过信息
- 如果只发现轻微问题（没有严重或中等问题）→ 输出审核结果，提示存在轻微问题
- 如果发现严重或中等问题 → 输出审核结果，等待用户修复后重新审核

## 错误修复原则

修复 lint 报错或代码问题时必须**正面解决问题**，不得回避：

- **不能**随意给变量添加 `any` 类型声明来绕过类型检查——这只是掩盖了潜在的类型错误
- **不能**删除未开发完毕的功能代码来消除报错——这会破坏用户的工作进度
- **应当**理解报错的实际含义，针对根本问题进行修复

## 并行任务时的例外

若当前有多个任务**正在并行执行**（多个子任务同时修改文件），**不要**在此期间执行 lint。

原因：lint 命令可能会自动修改本地代码（格式化等），若某个并行任务正在写入文件，可能导致文件内容异常或冲突。

等所有并行任务完成后，再统一执行一次 lint。

## 禁止主动执行的命令

- 编译命令、构建命令、部署命令
- 除 lint 相关命令（`npm run lint:fix` 或 `npm run lint`）外的 `package.json` 中定义的其他命令
- 除 `node --version` 外的其他 `npx` 命令
- 任何自动执行的测试命令

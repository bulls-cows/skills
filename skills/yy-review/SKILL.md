# Lint Check

执行代码质量检查，包括 Node 版本验证和 lint 检查。

## 功能特性

- 读取项目根目录下的 `.nvmrc` 文件获取要求的 Node.js 版本
- 检查当前 Node.js 版本是否不小于 `.nvmrc` 中记录的版本
- 如果版本满足要求，执行 `npm run lint` 进行代码风格和类型检查
- 如果 lint 出现错误，尝试自动修复

## 使用方式

直接调用 `/review` 即可执行代码质量检查。

## 工作流程

### 步骤 1：检查 Node.js 版本

1. 读取项目根目录下的 `.nvmrc` 文件，获取要求的 Node.js 版本
2. 执行 `node --version` 命令获取当前 Node.js 版本
3. 比较当前版本是否不小于 `.nvmrcrc` 中记录的版本

**情况 1：当前版本小于 `.nvmrc` 中的要求版本**

- 显示当前版本和要求版本信息
- 提示用户升级 Node.js 到要求的版本或更高版本
- 不执行 lint 检查

**情况 2：当前版本不小于 `.nvmrc` 中的要求版本**

- 继续执行步骤 2

### 步骤 2：执行 lint 检查

执行 `npm run lint` 命令检测代码风格和 TypeScript 类型。

**情况 1：lint 通过，无错误**

- 显示成功消息

**情况 2：lint 发现错误**

- 显示错误信息
- 尝试自动修复错误

### 错误修复原则

修复 lint 报错时必须**正面解决问题**，不得回避：

- **不能**随意给变量添加 `any` 类型声明来绕过类型检查——这只是掩盖了潜在的类型错误
- **不能**删除未开发完毕的功能代码来消除报错——这会破坏用户的工作进度
- **应当**理解报错的实际含义，针对根本问题进行修复

### 并行任务时的例外

若当前有多个任务**正在并行执行**（多个子任务同时修改文件），**不要**在此期间执行 lint。

原因：`npm run lint` 会自动修改本地代码（格式化等），若某个并行任务正在写入文件，可能导致文件内容异常或冲突。

等所有并行任务完成后，再统一执行一次 lint。

## 禁止主动执行的命令

- 编译命令、构建命令、部署命令
- 除 `npm run lint` 外的 `package.json` 中定义的其他命令
- 除 `node --version` 外的其他 `npx` 命令
- 任何自动执行的测试命令

